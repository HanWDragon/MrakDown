# 同步、异步、阻塞、非阻塞

- 同步和异步表现的是任务执行的方式
	- 按顺序执行
	- 异步的关键在于并行处理和不等待。
- 阻塞和非阻塞表现的是调用方的表现
	- 阻塞就是阻塞等待结果返回
	- 非阻塞就是继续执行
- 同步阻塞： 程序按顺序执行，每个任务完成后下一个任务才能开始，且在等待时阻塞。
- 同步非阻塞： 程序按顺序执行，但在任务完成前可以做其他事情。
- 异步阻塞： 任务可以在不依赖顺序的情况下启动，但仍然可能在等待某个任务时阻塞。
- 异步非阻塞： 任务可以在不依赖顺序的情况下启动，并且在等待时不阻塞，允许程序继续执行其他任务。
